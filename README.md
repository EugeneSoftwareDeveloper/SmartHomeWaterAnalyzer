# SmartHomeWaterAnalyzer

[![SmartHomeWaterAnalyzer-CheckIn](https://github.com/EugeneSoftwareDeveloper/SmartHomeService/actions/workflows/SmartHomeServiceCheckIn.yml/badge.svg)](https://github.com/EugeneSoftwareDeveloper/SmartHomeWaterAnalyzer/blob/master/.github/workflows/ci.yml)

Android-приложение для тестера качества воды **YINMIK BLE-C600** (и совместимых white-label вариантов семейства BLE-YC01) / 7 в 1 Temp ORP EC TDS минерализация S.G измеритель PH онлайн тестер качества воды с голубыми зубьями приложение для контроля питьевой лаборатории аквариума. 
Подключается к прибору по Bluetooth LE напрямую с телефона, читает текущие показания, отображает их с цветными шкалами «опасно / норма / отлично» — чтобы не сравнивать цифры с бумажной таблицей вручную.

## Что умеет

- **Сканирование BLE и подключение** к YINMIK BLE-C600. Фильтр по имени на клиенте + диагностика «BLE-устройств в эфире: N» + bottom-sheet со всеми видимыми BLE-устройствами для случаев, когда прибор называется неожиданно.
- **Чтение 7 параметров** с цветными шкалами, развёрнутыми названиями («Кислотность (pH)», «Минерализация (TDS)» и т.д.) и сводной оценкой воды.
- **Справка по каждому параметру** с тонкой градацией (5–9 уровней): тап по карточке → подробное описание с конкретными числовыми диапазонами и человеко-понятными заметками.
- **Управление прибором**: переключатели подсветки и HOLD (спекулятивные байты — см. инструкцию ниже).
- **Debug-страница для подбора команд**: 14 пресетов байтов + ручной hex-ввод, кнопка «Send & verify» проверяет, изменился ли целевой бит статуса. Открывается из шапки экрана показаний.
- **Локальная история измерений** в SQLite с **ручным сохранением** — FAB «Сохранить замер» на экране показаний; случайные/тестовые чтения историю не засоряют. Кнопка показывает состояние «Сохранено» с чек-иконкой; SnackBar даёт 5 секунд на отмену.
- **Метки замеров** (например, «Москва, квартира») — задаются полем на странице показаний, сохраняются ко всем последующим записям. Метка существующей записи редактируется из детального просмотра.
- **Хронология истории** — записи группируются по дню (Сегодня / Вчера / `dd.MM.yyyy`).
- **Удаление записей** — swipe-to-delete в списке с undo-SnackBar, либо «Удалить замер» в меню детального просмотра. Восстановление в течение 5 секунд возвращает запись с тем же id.
- **Детальный просмотр со свайпом** — тап по записи в истории показывает все карточки параметров, свайп влево/вправо листает между замерами для сравнения.
- **Графики во времени** — переключатель параметра над диаграммой (pH / ORP / EC / TDS / Salinity / Temp / S.G.), последние 50 точек, диапазон y-оси берётся из шкалы параметра.
- **Экспорт CSV** — вся история одним тапом, открывается через системный share-sheet.
- **Профили норм**: питьевая вода / бассейн / аквариум (пресный) / гидропоника — переключаются в настройках, влияют на цветные зоны и текст в справке.
- **Тёмная тема** — по системе, или принудительно в настройках.
- **Локальные уведомления** при выходе параметров из нормы (опционально).
- **Локализация**: русский + английский.
- **Реакция на выключение Bluetooth** — отдельный экран с кнопкой «Включить Bluetooth».
- **История доступна без подключения** — открывается из шапки главного экрана.

## Что показывает

7 параметров качества воды + статус прибора:

| Параметр | Единица | Что значит |
|---|---|---|
| **pH** | — | Кислотность/щёлочность. Норма питьевой воды 6.5–8.5 |
| **ORP** | мВ | Окислительно-восстановительный потенциал |
| **EC** | µС/см | Электропроводность |
| **TDS** | ppm | Общая минерализация |
| **Соленость** | ppm | Соленость в ppm |
| **Температура** | °C | Температура воды |
| **S.G.** | — | Удельная плотность |
| Батарея | мВ + % | Состояние батареи прибора (оценка) |

Каждый параметр — на отдельной карточке с **крупным значением** и **цветной шкалой**, на которой:
- цветные сегменты обозначают зоны качества для **питьевой воды** (ВОЗ/СанПиН-ориентир);
- треугольный маркер показывает текущее значение и **анимируется** при обновлении;
- бейдж в шапке карточки показывает текущую зону (`Опасно` / `Внимание` / `Приемлемо` / `Норма` / `Оптимум`);
- тап по карточке раскрывает описание и пояснение нормы.

Сверху — **сводная hero-карточка**: общая оценка воды (по худшему параметру) + батарея прибора + чипы HOLD/LIGHT, если активны на приборе.

Снизу — **управление**: переключатели подсветки и HOLD (см. [Управление](#управление-прибором) ниже).

## Требования

| Что нужно | Зачем |
|---|---|
| **Прибор YINMIK BLE-C600** или совместимый (BLE-YC01 family) | источник данных |
| **Android 7.0+** с поддержкой Bluetooth LE | целевая платформа |
| Закрытое официальное приложение YINMIK | прибор держит **только одно** BLE-подключение одновременно |

> **Важно про BLE-подключение**: BLE-C600 не позволяет двум клиентам подключиться к нему одновременно. Если у тебя на телефоне открыто официальное приложение YINMIK и оно подключено к прибору — наше приложение не найдёт его (или найдёт, но не сможет подключиться). Полностью закрой официальное приложение (force stop) перед использованием.

## Быстрый старт

### Установка APK из GitHub Releases

1. Открой страницу [Releases](https://github.com/EugeneSoftwareDeveloper/SmartHomeWaterAnalyzer/releases) и скачай APK для своей архитектуры:
   - **`water-analyzer-X.Y.Z-arm64-v8a.apk`** — большинство современных Android-телефонов (2018+). Если не уверен — бери этот.
   - `water-analyzer-X.Y.Z-armeabi-v7a.apk` — старые 32-битные устройства.
   - `water-analyzer-X.Y.Z-x86_64.apk` — для эмуляторов на ПК.
2. На телефоне разреши установку из неизвестных источников (Settings → Apps → Special access → Install unknown apps).
3. Установи APK, открой приложение.
4. Разреши Bluetooth и геолокацию (для BLE-сканирования на Android ≤11).
5. На приборе — длинное нажатие `ON/OFF` для включения Bluetooth-режима.
6. В приложении → «Сканировать» → выбери прибор → смотри показания.

> **Подпись APK**: если в репозитории не настроены release-secrets (см. ниже), релизные APK подписаны debug-keystore'ом. Это нормально для установки «скачал → поставил», но при апгрейде Android требует ту же подпись — устанавливай новые версии поверх старых без удаления.

## Использование

### 1. Подготовка прибора

Включи BLE на BLE-C600 длинным нажатием `ON/OFF` (до появления BLE-индикатора). Убедись, что **официальное приложение YINMIK не подключено** к прибору.

### 2. Сканирование

Открой Water Analyzer → автоматически начнётся сканирование. Прибор появится в списке как `BLE-C600` или похожее. Тапни на него.

При первом запуске Android попросит разрешения на Bluetooth (на 12+) или геолокацию (на старых версиях) — нужно разрешить.

### 3. Чтение показаний

После подключения сразу читается один кадр. На экране — hero-карточка с общей оценкой + 7 карточек параметров. Вытяни вниз (pull-to-refresh) или нажми ⟳ в правом верхнем углу для нового чтения.

### 4. Управление прибором

Внизу экрана — секция «Управление прибором» с переключателями:

- **Подсветка** — включает/выключает подсветку экрана прибора. Состояние отражает реальный бит 0x08 статус-байта FF02. После переключения приложение автоматически перечитывает кадр для подтверждения.
- **Удержание показаний (HOLD)** — фиксирует текущие значения на экране прибора. Бит 0x10 статус-байта.

> ⚠️ **Команды управления — спекулятивные.** Производитель не публикует протокол. Текущие байты в `lib/yinmik/commands.dart` — обоснованная догадка (single byte = бит статуса). На большинстве партий BLE-C600 это должно работать; если не работает на твоей — см. [docs/03-control-commands.md](./docs/03-control-commands.md), там пошаговая инструкция как добыть точные байты через HCI snoop log с официального приложения.

#### Production-signing через GitHub Secrets (опционально)

По умолчанию APK подписываются debug-keystore'ом, который генерируется в CI на лету. Этого достаточно для распространения через GitHub Releases, но Android при апгрейде требует **ту же** подпись, поэтому debug-keystore не подходит для долгой жизни приложения (CI генерирует разный ключ при каждом запуске).

Для стабильной production-подписи добавь четыре GitHub Secrets в репозитории (`Settings → Secrets and variables → Actions`):

| Secret | Что |
|---|---|
| `ANDROID_KEYSTORE_BASE64` | `base64 -w0 keystore.jks` — твой keystore.jks в base64. На Windows: `[Convert]::ToBase64String([IO.File]::ReadAllBytes("keystore.jks"))` |
| `ANDROID_KEYSTORE_PASSWORD` | Пароль от keystore |
| `ANDROID_KEY_ALIAS` | Имя ключа (например, `water-analyzer`) |
| `ANDROID_KEY_PASSWORD` | Пароль ключа |

Создание keystore (один раз):

```powershell
keytool -genkey -v -keystore water-analyzer-release.jks -keyalg RSA -keysize 2048 -validity 10000 -alias water-analyzer
```

После добавления secrets — все следующие релизы будут подписаны production-ключом. **Бэкап keystore сделай немедленно** — потеря ключа значит, что выпускать апгрейды с тем же `applicationId` уже не получится, и пользователям придётся удалить старую версию и поставить новую с нуля.

## Известные ограничения

- **Прибор держит только одно BLE-подключение** — официальное приложение YINMIK блокирует наше. Закрывай его перед использованием.
- **Команды управления спекулятивные** — могут не работать на конкретной партии. См. [docs/03-control-commands.md](./docs/03-control-commands.md) как это починить.
- **Battery percent — оценка** по формуле BLE-YC01 (1950–3190 mV → 0–100%). Для BLE-C600 точная калибровка не подтверждена.
- **Только Android** — Flutter-проект конфигурирован под `--platforms=android`. Архитектура совместима с iOS, но iOS-часть нужно доконфигурировать (Info.plist Bluetooth-разрешения).
- **Фильтр имени может не пропустить экзотическую партию** — ловим `BLE-C600` / `BLE_C600` / `Yinmik-*` / `*C600*` и пакеты с сервисом FF01. Если твой прибор называется как-то ещё, на экране сканирования нажми «Показать все устройства» и выбери вручную; заодно можно расширить `knownNameKeywords` в `lib/yinmik/client.dart`.
- **pH «дрожит»** — электрод стабилизируется во времени, мгновенные значения могут плавать на ±0.1. Это нормально.

## Связанный репозиторий

[`SmartHomeService`](https://github.com/EugeneSoftwareDeveloper/SmartHomeService) — .NET 10 сервис умного дома (Windows) с такой же интеграцией BLE-C600 на стороне сервера. Декодер портирован сюда один-в-один, эталонные тестовые кадры общие.

Архитектурно приложения **независимы**: SmartHomeWaterAnalyzer не требует SmartHomeService для работы. Можно использовать параллельно, но не на одном приборе одновременно (один BLE-канал).
