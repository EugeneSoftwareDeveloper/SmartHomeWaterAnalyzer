# SmartHomeWaterAnalyzer

Android-приложение для тестера качества воды **YINMIK BLE-C600** (и совместимых white-label вариантов семейства BLE-YC01). Подключается к прибору по Bluetooth LE напрямую с телефона, читает текущие показания, отображает их с цветными шкалами «опасно / норма / отлично» — чтобы не сравнивать цифры с бумажной таблицей вручную.

```
┌─────────────────────────────────────────────────────────┐
│ ◉  Хорошее качество воды                                │
│    Все измеренные параметры в пределах нормы.           │
│                                                         │
│    🔋 ~94%  (3015 мВ)              [LIGHT] [HOLD]      │
└─────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────┐
│ pH                                  ● Оптимум           │
│ 7.24                                                    │
│                                                         │
│ ████░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░                      │
│  ▼                                                      │
│ 0.0                                              14.0   │
└─────────────────────────────────────────────────────────┘
... (карточки для ORP, EC, TDS, Salinity, Temp, S.G.)

Управление прибором
┌─────────────────────────────────────────────────────────┐
│ 💡  Подсветка                                      [×]  │
│     Включить экран прибора                              │
├─────────────────────────────────────────────────────────┤
│ 🔒  Удержание показаний (HOLD)                     [○]  │
│     Зафиксировать текущие значения на экране            │
└─────────────────────────────────────────────────────────┘
```

## Что показывает

7 параметров качества воды + статус прибора:

| Параметр | Единица | Что значит |
|---|---|---|
| **pH** | — | Кислотность/щёлочность. Норма питьевой воды 6.5–8.5 |
| **ORP** | мВ | Окислительно-восстановительный потенциал |
| **EC** | µС/см | Электропроводность |
| **TDS** | ppm | Общая минерализация |
| **Соленость** | ppm | Соленость в ppm |
| **Температура** | °C | Температура воды |
| **S.G.** | — | Удельная плотность |
| Батарея | мВ + % | Состояние батареи прибора (оценка) |

Каждый параметр — на отдельной карточке с **крупным значением** и **цветной шкалой**, на которой:
- цветные сегменты обозначают зоны качества для **питьевой воды** (ВОЗ/СанПиН-ориентир);
- треугольный маркер показывает текущее значение и **анимируется** при обновлении;
- бейдж в шапке карточки показывает текущую зону (`Опасно` / `Внимание` / `Приемлемо` / `Норма` / `Оптимум`);
- тап по карточке раскрывает описание и пояснение нормы.

Сверху — **сводная hero-карточка**: общая оценка воды (по худшему параметру) + батарея прибора + чипы HOLD/LIGHT, если активны на приборе.

Снизу — **управление**: переключатели подсветки и HOLD (см. [Управление](#управление-прибором) ниже).

## Требования

| Что нужно | Зачем |
|---|---|
| **Прибор YINMIK BLE-C600** или совместимый (BLE-YC01 family) | источник данных |
| **Android 7.0+** с поддержкой Bluetooth LE | целевая платформа |
| Закрытое официальное приложение YINMIK | прибор держит **только одно** BLE-подключение одновременно |

> **Важно про BLE-подключение**: BLE-C600 не позволяет двум клиентам подключиться к нему одновременно. Если у тебя на телефоне открыто официальное приложение YINMIK и оно подключено к прибору — наше приложение не найдёт его (или найдёт, но не сможет подключиться). Полностью закрой официальное приложение (force stop) перед использованием.

## Быстрый старт

### Установка APK (если у тебя готовая сборка)

1. Скачай APK (например, из releases на GitHub).
2. На телефоне разреши установку из неизвестных источников (Settings → Apps → Special access → Install unknown apps).
3. Установи APK, открой приложение.
4. Разреши Bluetooth и геолокацию (для BLE-сканирования на Android ≤11).
5. На приборе — длинное нажатие `ON/OFF` для включения Bluetooth-режима.
6. В приложении → «Сканировать» → выбери прибор → смотри показания.

## Использование

### 1. Подготовка прибора

Включи BLE на BLE-C600 длинным нажатием `ON/OFF` (до появления BLE-индикатора). Убедись, что **официальное приложение YINMIK не подключено** к прибору.

### 2. Сканирование

Открой Water Analyzer → автоматически начнётся сканирование. Прибор появится в списке как `BLE-C600` или похожее. Тапни на него.

При первом запуске Android попросит разрешения на Bluetooth (на 12+) или геолокацию (на старых версиях) — нужно разрешить.

### 3. Чтение показаний

После подключения сразу читается один кадр. На экране — hero-карточка с общей оценкой + 7 карточек параметров. Вытяни вниз (pull-to-refresh) или нажми ⟳ в правом верхнем углу для нового чтения.

### 4. Управление прибором

Внизу экрана — секция «Управление прибором» с переключателями:

- **Подсветка** — включает/выключает подсветку экрана прибора. Состояние отражает реальный бит 0x08 статус-байта FF02. После переключения приложение автоматически перечитывает кадр для подтверждения.
- **Удержание показаний (HOLD)** — фиксирует текущие значения на экране прибора. Бит 0x10 статус-байта.

> ⚠️ **Команды управления — спекулятивные.** Производитель не публикует протокол. Текущие байты в `lib/yinmik/commands.dart` — обоснованная догадка (single byte = бит статуса). На большинстве партий BLE-C600 это должно работать; если не работает на твоей — см. [docs/03-control-commands.md](./docs/03-control-commands.md), там пошаговая инструкция как добыть точные байты через HCI snoop log с официального приложения.

## Сборка

### Что нужно установить

1. [Flutter SDK 3.41+](https://docs.flutter.dev/get-started/install/windows) (вместе тянет Dart SDK)
2. Android SDK (через Android Studio или standalone command-line tools)
3. Включённый «режим разработчика» и USB-debugging на телефоне (или Android-эмулятор с BLE)

Проверь готовность:

```powershell
flutter doctor
```

### Dev-запуск (с подключённым телефоном)

```powershell
cd D:\AI\SmartHomeWaterAnalyzer
flutter pub get
flutter run
```

Hot reload работает: правки в Dart-коде применяются без переустановки.

### Release APK

```powershell
flutter build apk --release
```

Готовый файл — `build/app/outputs/flutter-apk/app-release.apk`. Скопируй на телефон и установи.

### Тесты и анализ


## Известные ограничения

- **Прибор держит только одно BLE-подключение** — официальное приложение YINMIK блокирует наше. Закрывай его перед использованием.
- **Команды управления спекулятивные** — могут не работать на конкретной партии. См. [docs/03-control-commands.md](./docs/03-control-commands.md) как это починить.
- **Зоны качества для питьевой воды** — нормы для бассейна / аквариума другие. Профили норм запланированы, но не реализованы.
- **Battery percent — оценка** по формуле BLE-YC01 (1950–3190 mV → 0–100%). Для BLE-C600 точная калибровка не подтверждена.
- **Только Android** — Flutter-проект конфигурирован под `--platforms=android`. Архитектура совместима с iOS, но iOS-часть нужно доконфигурировать (Info.plist Bluetooth-разрешения).
- **Без истории и графиков** — каждое чтение замещает предыдущее. История — V2.
- **pH «дрожит»** — электрод стабилизируется во времени, мгновенные значения могут плавать на ±0.1. Это нормально.

## Что планируется

- Снять HCI snoop log с официального приложения и заменить спекулятивные байты команд на реальные.
- Локальная история измерений в SQLite + графики (`fl_chart`).
- Дополнительные профили норм: бассейн (хлорированная вода), аквариум (пресная/морская), гидропоника.
- Тёмная тема.
- iOS-сборка.
- Push-уведомления при выходе параметров из нормы.
- Экспорт измерений в CSV.

## Связанный репозиторий

[`SmartHomeService`](https://github.com/EugeneSoftwareDeveloper/SmartHomeService) — .NET 10 сервис умного дома (Windows) с такой же интеграцией BLE-C600 на стороне сервера. Декодер портирован сюда один-в-один, эталонные тестовые кадры общие.

Архитектурно приложения **независимы**: SmartHomeWaterAnalyzer не требует SmartHomeService для работы. Можно использовать параллельно, но не на одном приборе одновременно (один BLE-канал).
