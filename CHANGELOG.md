## [1.1.0] - 2026-05-25

### Добавлено
- **Ручное сохранение замеров в историю** (FAB «Сохранить замер» на ReadingPage). Раньше каждый успешный `_refresh` автоматически писал в БД — теперь история чище, тестовые/калибровочные замеры её не засоряют. Кнопка сразу после сохранения превращается в `Сохранено` + чек-иконку, повторный тап заблокирован пока не появится новый кадр. SnackBar показывает «Замер сохранён» с действием «Отменить» (5 секунд → `repository.deleteById(id)`).
- **Удаление записей истории**:
  - В `HistoryPage` — swipe-to-delete с красной зоной «Удалить» (Material 3 `errorContainer`). После свайпа — SnackBar undo 5 секунд через `restoreFromMeasurement(row)` (восстанавливается с тем же id для сохранения индексов в PageView).
  - В `HistoryDetailPage` — ⋮-меню → «Удалить замер» с подтверждением. После удаления PageView сдвигается на соседнюю запись; если удалили последнюю — экран автоматически закрывается.
- **Редактирование метки замера**: в HistoryDetailPage ⋮-меню → «Изменить метку» открывает диалог с TextField. Пустая строка = метки нет. Update точечный через `repository.updateLabel(id, label)` — без переписывания всей записи.
- **Графики для всех параметров, не только pH**: над `LineChart` в `HistoryPage` появился `DropdownButton` с выбором параметра (pH/ORP/EC/TDS/Salinity/Temp/S.G.). y-axis range и шаг сетки подбираются автоматически из `WaterParameter.scaleMin/scaleMax`; для больших значений (TDS, EC) шкала компактная — «1.5k» вместо «1500».
- **Хронология истории**: записи группируются по дню («Сегодня», «Вчера», или `dd.MM.yyyy` для более старых) с разделителями-заголовками. Раньше был один длинный список — для двухнедельной истории терялся контекст «когда это было».
- **Развёрнутые названия параметров**: «Кислотность (pH)», «Редокс-потенциал (ORP)», «Электропроводность (EC)», «Минерализация (TDS)», «Солёность», «Температура», «Плотность воды (S.G.)». У `WaterParameter` появилось поле `displayLabel` = «название (короткая_аббревиатура)» для шапок карточек.
- **Справка по параметрам** (`lib/help/parameter_help.dart`) с тонкой градацией зон — 5–9 уровней (вместо 4–5 в основной шкале) с конкретными числовыми границами и человеко-понятными заметками. Доступна из шапки главного экрана, настроек, и тапом по любой карточке параметра.
- **Именованные замеры**: поле «Метка замера» на странице показаний (например, «Москва, квартира»). Метка сохраняется в `AppSettings.currentLabel` и применяется ко всем последующим записям истории. В истории отображается под временем с иконкой `label_outline`.
- **История без подключения**: иконка `history` в шапке HomePage открывает `/history` отдельным маршрутом — историю можно смотреть без живого BLE-подключения.
- **Детальный просмотр замера со свайпом**: тап по записи в истории открывает `HistoryDetailPage` с теми же карточками, что и на основном экране. Свайп влево/вправо переключает между соседними замерами для сравнения. В шапке — номер и общее число записей.
- **Debug-страница команд** (`lib/ui/debug_commands_page.dart`) — открывается из шапки страницы показаний:
  - Выбор UUID характеристики (FF15 / FF02)
  - Выбор бита для проверки (0x08 / 0x10)
  - 14 готовых пресетов байтов + ручной ввод hex
  - «Send & verify»: записывает команду, перечитывает кадр FF02, показывает изменился ли целевой бит
  - Лог попыток с цветной индикацией (✓ зелёная при успехе, ✗ серая, ❌ красная), кнопка очистки лога
- **Сводная диагностика сканирования**: на экране поиска отображается «BLE-устройств в эфире: N»; после окончания сканирования — кнопка «Показать все устройства» открывает bottom-sheet со всеми BLE-устройствами без фильтра по имени.
- **Тёмная тема** (Material 3 с тем же seed-цветом для светлой и тёмной).
- **Локализация**: русский (основной) + английский (через ARB-файлы в `lib/l10n/`).
- **Профили норм качества воды**: питьевая, бассейн, аквариум (пресный), гидропоника — переключаются в настройках. Влияют на пороги зон и текст в справке.
- **История измерений** в локальной SQLite-БД через drift. Schema v2 с колонкой `label`.
- **Экспорт истории в CSV** + системный share-sheet через `share_plus`.
- **Локальные уведомления** при выходе параметров из нормы (опционально включаются в настройках). Триггерятся в `_refresh`, не в `build`, чтобы не дублироваться при ребилдах.
- **Запоминание последнего подключённого устройства** в `SharedPreferences`.
- **Bottom navigation**: Показания / История / Настройки.
- **Глобальный error handler** через `FlutterError.onError` + `PlatformDispatcher.onError`.
- **Реакция UI на выключение Bluetooth** — отдельный экран «Bluetooth выключен» + кнопка «Включить Bluetooth» (программно через `FlutterBluePlus.turnOn()`).
- **Haptic feedback** на переключателях управления и при ошибках.
- **BLE retry**: 3 попытки с 2-секундной паузой на транзиентных сбоях.
- **MTU negotiation** (request 247) при подключении.
- **Strict-mode статический анализатор**: `prefer_single_quotes`, `require_trailing_commas`, `avoid_dynamic_calls`, `cancel_subscriptions`, `only_throw_errors`, `strict-casts`, `strict-inference` и др.
- **Release signing scaffold** через `android/key.properties` + fallback на debug-ключ.
- **R8 минификация + shrinkResources** + ProGuard-правила для рефлекшен-критичных пакетов (BLE, drift, notifications).
- **Split-per-ABI APK**: одна сборка → три APK по архитектурам (~18–22 MB каждый вместо одного 60 MB).
- **GitHub Actions CI**: lint + test + сборка APK артефакта.

### Изменено
- **UI на `flutter_riverpod`**: providers вместо singleton'ов, `ConsumerWidget`/`ConsumerStatefulWidget` везде, где нужна реактивность.
- **Навигация на `go_router`** с маршрутами `/`, `/device`, `/history`, `/history/detail`, `/help`, `/debug-commands`.
- **`WaterParameterCatalog.forProfile(profile)`** возвращает разные зоны качества в зависимости от выбранного профиля.
- **`YinmikBleClient`** инкапсулирует retry, MTU, корректную отписку сканера; возвращает `ScanState` с количеством общих видимых устройств для диагностики.
- **`_extractValues`** вынесен в общий хелпер `lib/yinmik/reading_values.dart` — раньше дублировался в `reading_page.dart` и `history_detail_page.dart`.
- **`HistoryRepository.save` теперь возвращает `Future<int>`** (id вставленной строки) — нужен для undo в кнопке «Сохранить» и swipe-to-delete. Добавлены методы `updateLabel(id, label)`, `deleteById(id)`, `restoreFromMeasurement(m)`.
- **`onReadingUpdated` в `ControlPanel`** больше не сохраняет автоматически после переключения подсветки/HOLD — только обновляет `_reading` в стейте. Если пользователь хочет зафиксировать состояние «прибор с включенной подсветкой» в истории, он явно нажимает FAB.
- **`_MeasurementTile`** теперь показывает только время (`HH:mm`) вместо `dd.MM HH:mm` — дата вынесена в групповой заголовок выше, повторение избыточно.
- **Чистая логика вынесена из UI в testable helpers**:
  - `groupMeasurementsByDay` — отдельный файл `lib/history/grouping.dart` с опциональным параметром `now` для детерминированных тестов.
  - `niceAxisInterval` + `formatChartAxisLabel` — `lib/ui/widgets/chart_axis.dart`.
  - `AppDatabase.forTesting(QueryExecutor)` — конструктор для in-memory drift в тестах.
- **Test suite: 20 → 50 тестов**. Новые файлы:
  - `test/chart_axis_test.dart` — 9 тестов на подбор шага и формата меток оси y.
  - `test/measurement_grouping_test.dart` — 8 тестов на группировку «Сегодня»/«Вчера»/`dd.MM.yyyy`, включая границу 23:59→00:00.
  - `test/history_repository_test.dart` — 13 тестов на CRUD через `AppDatabase.forTesting(NativeDatabase.memory())`: save→id, updateLabel с trim, deleteById, restoreFromMeasurement сохраняет исходный id, watchRecent стримит изменения.

### Исправлено
- **CI сборка APK падала** с exit code 1: `flutter build apk --release` без `--split-per-abi` не создаёт fat-APK, потому что в `android/app/build.gradle.kts` включен `splits.abi` с `isUniversalApk = false`. Файл `app-release.apk` отсутствовал → upload-artifact падал «no files found». Теперь CI использует `--split-per-abi` и upload через wildcard `app-*-release.apk`.
- **GitHub Releases — автоматическая публикация**: добавлен workflow [`release.yml`](.github/workflows/release.yml). Триггер — push тега `v*`. Собирает три APK (один на ABI), переименовывает в `water-analyzer-X.Y.Z-<abi>.apk` и создаёт GitHub Release с описанием из соответствующего раздела `CHANGELOG.md`. Поддерживает production-signing через secrets `ANDROID_KEYSTORE_BASE64`/`ANDROID_KEYSTORE_PASSWORD`/`ANDROID_KEY_ALIAS`/`ANDROID_KEY_PASSWORD`; без них собирается с debug-keystore. Подробности — в README, секция «Выпуск релиза».
- **Фильтр имени слишком строгий**: на некоторых партиях BLE-C600 объявляет имя как `BLE_C600` (с подчёркиванием), `Yinmik-C600-XXXX`, или вовсе пустое имя в advertisement. Прежний фильтр `startsWith('BLE-C600' | 'BLE-YC' | 'YINMIK')` такие приборы отсекал. Теперь:
  - `contains` по расширенному списку ключевых слов (`BLE-C600`, `BLE_C600`, `BLE C600`, `BLE-YC`, `BLE_YC`, `YINMIK`, `YC01`, `C600`);
  - проверка как `platformName`, так и `advertisementData.advName`;
  - fallback: пакеты с сервисом FF01 в advertisement принимаются даже без имени.
- **Stop → Start больше не «замораживает» сканер**: при нажатии Stop в шапке прежний код звал только `FlutterBluePlus.stopScan()`, не отменяя подписку. Platform-сканер оставался в полу-остановленном состоянии, новый `startScan` тихо игнорировался и `BLE-устройств в эфире` оставалось 0 до полной перезагрузки приложения. Теперь `_stopScan()` сначала отменяет `StreamSubscription`, что прогоняет `finally` в `YinmikBleClient.scan` и освобождает keepAlive-listener. Перед новым `startScan` добавлена пауза 300 мс на обработку stop платформой.
- **«Показать все устройства» доступно во время сканирования**: раньше кнопка появлялась только после окончания скана. Теперь — сразу как только сканер увидел хоть одно устройство, чтобы можно было выбрать прибор вручную, не дожидаясь таймаута.
- **Регрессия: устройство не находилось в скане** после введения `withServices: [serviceUuid]` платформенного фильтра. BLE-C600 не объявляет сервис FF01 в advertisement, поэтому ОС-стек его отбрасывал. Возвращена клиентская фильтрация по имени.
- **Регрессия: пустой стрим scanResults** — восстановлен «прогревающий» `subscription = FlutterBluePlus.scanResults.listen((_) {})` перед `startScan`. Без него `await for` мог не получать результатов в некоторых версиях flutter_blue_plus.
- **Permission check для Android 12+**: больше не требует геолокацию, корректно различает Android ≤11 и 12+. UI показывает конкретную причину отказа.
- **`DateFormat('dd MMM yyyy, HH:mm', 'ru')`** заменён на локаль-нейтральный `'dd.MM.yyyy HH:mm'` — раньше мог упасть на устройствах без российских locale data.
- **Уведомления о выходе из нормы** перенесены из `build` в `_refresh` — раньше могли дублироваться при ребилдах виджета.
